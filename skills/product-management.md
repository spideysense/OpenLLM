# Skill: Product Management

## Product Requirements Document (PRD)
When asked to write a PRD, use this structure:
```
# [Feature Name]
## Problem Statement
What user pain does this solve? Include data/evidence.
## Goals & Success Metrics
- Primary metric (e.g., 20% increase in activation)
- Secondary metrics
- Guardrail metrics (what should NOT decrease)
## User Stories
As a [persona], I want to [action] so that [outcome].
## Scope
### In Scope (v1)
### Out of Scope (future)
## Design
Link to designs or describe the UX flow step-by-step.
## Technical Approach
High-level architecture. What systems are affected?
## Risks & Mitigations
## Timeline
| Phase | Duration | Deliverable |
## Open Questions
```

## Prioritization Frameworks
- **RICE**: Reach × Impact × Confidence / Effort
- **ICE**: Impact × Confidence × Ease (1-10 each)
- **MoSCoW**: Must have / Should have / Could have / Won't have
- **Kano Model**: Basic → Performance → Delight features

## Roadmap
When creating a roadmap:
- Group by quarter or month
- Use themes (not feature lists): "Onboarding Experience" not "Add tooltip"
- Include: Theme, Key deliverables, Success metric, Dependencies
- Format as a table or timeline

## User Research
- **Jobs to Be Done**: "When [situation], I want to [motivation] so I can [outcome]"
- **5 Whys**: dig to root cause of user problems
- **User Journey Map**: Awareness → Consideration → Activation → Retention → Referral

## Metrics
- **North Star Metric**: one metric that captures core value delivery
- **AARRR Funnel**: Acquisition → Activation → Retention → Revenue → Referral
- **Leading vs Lagging**: leading indicators predict, lagging confirm

## Competitive Analysis
| Feature | Us | Competitor A | Competitor B |
Use: ✅ (have), 🔶 (partial), ❌ (missing), 🏆 (best-in-class)

## Go-to-Market
- Target segment and persona
- Positioning statement: "For [target] who [need], [product] is a [category] that [benefit]. Unlike [competitor], we [differentiator]."
- Launch channels, messaging, success criteria
